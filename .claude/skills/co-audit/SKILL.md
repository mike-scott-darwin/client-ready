# /co-audit — Vault Health Check

Scans your entire vault and reports on context health. Identifies what needs attention so your AI outputs stay sharp.

## Usage

```
/co-audit               → Full vault audit
/co-audit context       → Audit Context files only
/co-audit stale         → Show files not updated in 14+ days
/co-audit orphans       → Show files with no cross-references
```

## Procedure

### 1. Scan All Files

Read every file in:
- `core/` — Core identity
- `decisions/` — Strategic decisions
- `research/` — Market intelligence
- `campaigns/` — Generated content
- `log/` — Daily logs

### 2. Assess Each Dimension

**Depth Score (per file) — read from `status` in frontmatter:**
- Empty — no status or template only
- Getting there — `status: draft` (extracted, not yet enriched)
- Solid — `status: active` (enriched at least once)
- Deep — `status: compounding` (enriched + cross-referenced)

**Freshness Score:**
- Fresh: updated within 7 days
- Current: updated within 14 days
- Stale: not updated in 14-30 days
- Overdue: not updated in 30+ days

**Connection Score:**
- Count `[[links]]` in and out
- Flag files with zero connections (orphans)
- Flag Context files that reference each other (good)

**Completeness Score:**
- Check each Context file for empty sections (marked with `<!-- -->` comments)
- Calculate percentage of sections filled

**Cron health (operator-run, if Hermes is configured):**
- Run `bin/cron-audit.sh` (reads `hermes cron list --all`). It flags cron accumulation over the cap, schedule stampedes, jobs drifted out of the governed set, and last-run errors — the weekly check that keeps the 2026-06-06 runaway from recurring. Roll its `~/.codify/cron-status` and findings into the report. Skip silently if `hermes` isn't on PATH (client-side vaults have no crons).

### 3. Generate Report

Output a structured report:

```
## Vault Audit — [date]

### Overall Context Power: [score]/100

### Context Files
| File | Depth | Freshness | Connections | Completeness |
|------|-------|-----------|-------------|--------------|
| soul.md | Working | Fresh | 3 in / 2 out | 85% |
| audience.md | Draft | Stale | 0 in / 1 out | 40% |
| ... | ... | ... | ... | ... |

### Action Items (Priority Order)
1. [Most impactful enrichment opportunity]
2. [Second priority]
3. [Third priority]

### Compound Score Trend
- Total vault files: [n]
- Total cross-references: [n]
- Files at Working+ depth: [n/total]
- Decisions logged this month: [n]
```

### 4. Save Report

Write to `campaigns/[date]-vault-audit.md` for historical tracking.

### 5. Generate Visual Dashboard

Also save an HTML dashboard to `campaigns/vault-dashboard.html`. This is a single self-contained file the client can open in any browser.

**Design:**
- Dark background (#0f172a), clean typography
- Overall score as a large animated gauge (0-100)
- Four smaller gauges: Depth, Freshness, Connections, Completeness
- Colour coding: green (80+), amber (50-79), red (below 50)
- File list table with status badges (colour-coded pills)
- Action items section with numbered priorities
- Footer: "Generated by Codify · [date]"

**Rules:**
- All CSS inline in a `<style>` block. No external dependencies.
- Use CSS animations for gauge fills (smooth on load)
- Mobile-responsive
- No JavaScript unless needed for interactivity
- Use real data from the audit — not placeholder values
- Keep it clean. No gradients, no neon, no sci-fi gimmicks. Professional.

**Why this exists:** Clients screenshot this and share it. It's proof the vault is working. Every share is a referral.

## Brain Sync Input — `/insights` (Architect-Side)

During monthly Brain Sync sessions, the architect should also run Claude Code's built-in `/insights` command against the client's session history:

```bash
claude -p "/insights"
```

This outputs an HTML report at `~/.claude/usage-data/report.html` with:
- Top friction sources (misunderstood requests, wrong-approach incidents)
- Suggested CLAUDE.md directives specific to the client's patterns
- Recommended skill additions based on observed usage
- Inferred-satisfaction distribution

**What to do with the output:**
1. Copy the HTML to `campaigns/[date]-claude-insights.html` in the client's vault.
2. Read the "Suggested CLAUDE.md Additions" section — these are usually worth adding to the client's CLAUDE.md verbatim.
3. Surface any recurring friction patterns in the Brain Sync call. These are the places Claude is failing the client; they're also the places the vault terminology or skill surface needs tightening.
4. If `/insights` suggests a skill we don't ship yet (e.g., `/co-deploy`, `/status`), consider whether it belongs in the vault template or in a client-specific skills folder.

**Why this matters:** `/insights` surfaces friction the client may not have articulated. It's the diagnostic layer that turns a Brain Sync from a status update into a targeted tuning session.
