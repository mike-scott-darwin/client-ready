# Checkout Setup Guide

**Sprint Module 2.3 — Estimated time: 1-2 hours**

---

## How to Use This

Step-by-step instructions for setting up your checkout page in GoHighLevel (GHL). Follow in order. If you get stuck, screenshot the issue and post it in the WhatsApp group.

---

## Prerequisites Checklist

Before you start, make sure you have:

- [ ] GHL account active (you should have received login details)
- [ ] Stripe account connected to GHL (Settings → Payments → Stripe)
- [ ] Your offer name, price, and description ready (from Module 1.3)
- [ ] At least one bump offer idea (we'll brainstorm more below)

---

## Step-by-Step Checkout Setup

### Step 1: Create Your Product

1. Go to **Payments → Products**
2. Click **Create Product**
3. Enter your product name
4. Set the price (one-time payment)
5. Add a short description
6. Save

### Step 2: Create Your Funnel

1. Go to **Sites → Funnels**
2. Click **Create New Funnel**
3. Name it something clear (e.g., "Client Ready Funnel")
4. Add these pages in order:
   - Landing page (opt-in or sales page)
   - Checkout page (order form)
   - Thank you page (confirmation)

### Step 3: Set Up the Checkout Page

1. Click on your checkout page in the funnel builder
2. Add an **Order Form** element
3. Connect it to your Stripe product
4. Configure the fields:
   - Name (required)
   - Email (required)
   - Payment info (auto from Stripe)
5. Set the submit button text (e.g., "Get Instant Access")

### Step 4: Add Your Order Bump(s)

1. In the Order Form settings, find **Order Bumps**
2. Click **Add Bump**
3. Select or create the bump product
4. Write the bump headline (keep it under 10 words)
5. Write the bump description (under 50 words — see templates below)
6. Set the bump price
7. Toggle bump display to ON

### Step 5: Configure the Thank You Page

1. Click on your thank you page
2. Add a clear message: "You're in. Check your email for access."
3. Include what to expect next (delivery timeline, email to watch for)
4. Optional: add a video welcome message

### Step 6: Connect Automation

1. Go to **Automations → Create Workflow**
2. Set trigger: **Payment Received** (or **Order Form Submitted**)
3. Add action: **Send Email** (your Day 1 welcome email)
4. Add action: **Add to Campaign** (your 10-day welcome sequence)
5. Add action: **Add Tag** (e.g., "purchased-[product-name]")
6. Save and activate

### Step 7: Test the Purchase Flow

1. Set Stripe to **test mode** (Settings → Payments → toggle test mode)
2. Go through your entire funnel as a buyer
3. Verify:
   - [ ] Landing page loads correctly
   - [ ] Checkout page displays product and price
   - [ ] Bump offer appears and can be added
   - [ ] Test payment goes through
   - [ ] Thank you page displays
   - [ ] Welcome email arrives
   - [ ] Automation triggers correctly
4. Switch Stripe back to **live mode** when done testing

## Order Bump Brainstorming

### The 3 Types of Bumps That Work

**Type 1: Quick Win**
Something they can use today. Immediate value.
- Example: DM scripts, templates, checklists, swipe files
- Works because: reduces the gap between buying and getting a result

**Type 2: Deeper Dive**
Goes further into one aspect of your main offer.
- Example: Advanced training on one topic, extra modules, case studies
- Works because: buyers who want more will pay for depth

**Type 3: Done-For-You Element**
Skips a step for them.
- Example: Pre-built templates, custom review, setup guide
- Works because: time is their scarcest resource

### Bump Description Formula

**[Result] + [How] + [Timeframe]** — in under 50 words.

**Example 1 (Quick Win):**
"Start sales conversations today. 5 copy-paste DM scripts for warm outreach — the reconnection, the value drop, the soft pitch, and two follow-ups. Send your first DM in 5 minutes."

**Example 2 (Deeper Dive):**
"Close without feeling salesy. 5 conversation frameworks for turning interest into a sale, plus word-for-word objection responses. Works in DMs, on calls, or in person."

**Example 3 (Done-For-You):**
"Skip the blank page. Complete offer document template, landing page swipe files, and a 30-day email sequence — all pre-written. Plug in your offer and go."

## Checkout Optimization Checklist

Before going live, verify:

- [ ] Product name and price are correct
- [ ] Bump offer displays clearly with its description
- [ ] Guarantee is mentioned on the checkout page (badge or text)
- [ ] "Secure checkout" or trust language is visible
- [ ] No external links or navigation that could distract
- [ ] Page loads correctly on mobile
- [ ] If you have testimonial snippets, add 1-2 above the bumps
- [ ] Button text is action-oriented ("Get Instant Access" not "Submit")
- [ ] Test purchase completed successfully in test mode
